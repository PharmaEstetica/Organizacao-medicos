3. Está usando dados vazios em vez dos dados salvos

## Possíveis Causas e Soluções

### Causa 1: Pedidos não estão sendo salvos com report_id
```typescript
// Ao processar CSV, verificar se está salvando o report_id:

// ERRADO (sem report_id):
await db.run(`
  INSERT INTO csv_orders (prescriber_id, order_date, status, net_value, patient)
  VALUES (?, ?, ?, ?, ?)
`);

// CORRETO (com report_id):
await db.run(`
  INSERT INTO csv_orders (report_id, prescriber_id, order_date, status, net_value, patient)
  VALUES (?, ?, ?, ?, ?, ?)
`, [reportId, prescriberId, ...]);
```

### Causa 2: Buscando da tabela errada
```typescript
// ERRADO (tabela errada):
const orders = await db.all('SELECT * FROM orders WHERE report_id = ?', [id]);

// CORRETO (tabela certa):
const orders = await db.all('SELECT * FROM csv_orders WHERE report_id = ?', [id]);
```

### Causa 3: Filtro de status errado
```typescript
// ERRADO (comparando com valor errado):
const efetivados = orders.filter(o => o.status === 'Aprovado');

// CORRETO (comparando com valor salvo):
const efetivados = orders.filter(o => o.status === 'Efetivado');

// OU buscar direto do banco:
const efetivados = await db.all(`
  SELECT * FROM csv_orders 
  WHERE report_id = ? AND status = 'Efetivado'
`, [reportId]);
```

### Causa 4: PDF usando dados do relatório em vez dos pedidos
```typescript
// ERRADO (usando só totais salvos, sem pedidos):
generatePDF({
  effectiveOrders: [],  // VAZIO!
  totalEffective: report.total_effective_value,  // Valor existe mas sem detalhes
});

// CORRETO (buscando pedidos reais):
const orders = await db.all('SELECT * FROM csv_orders WHERE report_id = ?', [id]);
generatePDF({
  effectiveOrders: orders.filter(o => o.status === 'Efetivado'),
  // ...
});
```
