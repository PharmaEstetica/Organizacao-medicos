# AJUSTE: Senha para Apagar Fórmulas

## O que fazer

Ao clicar no ícone de lixeira (🗑️) para excluir uma fórmula, deve abrir o modal de senha antes de executar a exclusão.

Usar o mesmo sistema de senha já configurado em **Configurações** → **"Excluir Cadastros/Fórmulas"**.

---

## Fluxo
```
1. Usuário clica no 🗑️ de uma fórmula
              ↓
2. Sistema verifica se "delete_protected" está ativo
              ↓
   Se NÃO → Pergunta confirmação normal e exclui
   Se SIM ↓
              ↓
3. Abre modal de senha: "Digite a senha para excluir"
              ↓
4. Usuário digita senha
              ↓
5. Sistema valida contra "delete_password" das configurações
              ↓
   Senha errada → Mostra erro, não exclui
   Senha certa → Exclui a fórmula
```

---

## Implementação

### No componente de lista de fórmulas:
```tsx
// FormulaList.tsx ou similar

import { useState } from 'react';
import { Trash2 } from 'lucide-react';
import { PasswordModal } from '../components/PasswordModal';

function FormulaList({ formulas, onDelete }) {
  const [showPasswordModal, setShowPasswordModal] = useState(false);
  const [formulaToDelete, setFormulaToDelete] = useState<number | null>(null);
  const [deleteProtected, setDeleteProtected] = useState(true);

  // Carregar configuração ao montar
  useEffect(() => {
    fetch('/api/settings')
      .then(res => res.json())
      .then(data => {
        setDeleteProtected(data.delete_protected);
      });
  }, []);

  const handleDeleteClick = (formulaId: number) => {
    setFormulaToDelete(formulaId);
    
    if (deleteProtected) {
      // Senha ativada → abre modal
      setShowPasswordModal(true);
    } else {
      // Senha desativada → confirmação simples
      if (confirm('Tem certeza que deseja excluir esta fórmula?')) {
        executeDelete(formulaId);
      }
    }
  };

  const handlePasswordVerify = async (password: string): Promise<boolean> => {
    const res = await fetch('/api/settings/verify', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ area: 'delete', password })
    });

    if (res.ok && formulaToDelete) {
      await executeDelete(formulaToDelete);
      setShowPasswordModal(false);
      setFormulaToDelete(null);
      return true;
    }
    return false;
  };

  const executeDelete = async (formulaId: number) => {
    await fetch(`/api/formulas/${formulaId}`, { method: 'DELETE' });
    onDelete(formulaId); // Atualiza lista
  };

  return (
    <>
      {formulas.map((formula) => (
        <div key={formula.id} className="...">
          {/* ... conteúdo do card ... */}
          
          <button
            onClick={() => handleDeleteClick(formula.id)}
            className="p-2 text-gray-500 hover:text-red-600 hover:bg-red-50 rounded"
            title="Excluir fórmula"
          >
            <Trash2 size={18} />
          </button>
        </div>
      ))}

      <PasswordModal
        isOpen={showPasswordModal}
        onClose={() => {
          setShowPasswordModal(false);
          setFormulaToDelete(null);
        }}
        onVerify={handlePasswordVerify}
        title="Digite a senha para excluir esta fórmula"
      />
    </>
  );
}
```

## Verificar se já existe

A configuração **"Excluir Cadastros/Fórmulas"** já deve existir em:
```sql
-- Tabela settings
setting_key: 'delete_protected'    → 'true' ou 'false'
setting_key: 'delete_password'     → senha configurada
```

Se não existir, adicionar:
```sql
INSERT INTO settings (setting_key, setting_value) VALUES
  ('delete_protected', 'true'),
  ('delete_password', 'excluir123');
```

## Aplicar em todos os lugares com exclusão

| Local | Tipo | Pedir senha? |
|-------|------|--------------|
| Lista de Fórmulas | 🗑️ Excluir fórmula | ✅ SIM |
| Lista de Cadastros (Médicos) | 🗑️ Excluir médico | ✅ SIM |
| Lista de Embalagens | 🗑️ Excluir embalagem | ✅ SIM |
| Documentos anexados | ✕ Remover documento | ❌ NÃO (só remove anexo) |

## Resumo

- Usar o modal `PasswordModal` que já existe
- Usar a rota `/api/settings/verify` com `area: 'delete'`
- Respeitar o toggle ON/OFF das configurações
- Se toggle OFF → confirmação simples sem senha
- Se toggle ON → exige senha antes de excluir
